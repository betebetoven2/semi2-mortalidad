# Entregables y Criterios de Aceptación

## Entregables de Fase 3

De acuerdo con el enunciado, la Fase 3 debe dejar evidencia de:

- modelo de ML en Amazon SageMaker o Databricks,
- dos visualizaciones en Power BI,
- dos visualizaciones en otra herramienta,
- análisis pre/post-COVID,
- documentación desplegada con MkDocs,
- y un flujo demostrable de extremo a extremo.

## Productos técnicos esperados

- repositorio GitHub con pipelines versionados,
- arquitectura de despliegue en Draw.io,
- modelo de datos y ERD,
- DW nube y local interoperables,
- modelo de ML con artefacto reproducible,
- archivo `.pbix`,
- y reporte de consultoría en formato Word o PDF.

## Productos de gestión esperados

Además del componente técnico, el enunciado solicita:

- Project Charter,
- Gantt de 4 meses,
- WBS,
- matriz RACI,
- gestión de riesgos,
- y plan de aseguramiento de calidad.

## Criterio rector

La fase no se evalúa como un incremento aislado. Debe poder demostrarse el recorrido completo del dato desde la fuente hasta el resultado analítico final.

## Relación con Fase 2

Esta sección no reemplaza la documentación de Stage. Stage sigue siendo la documentación de conformación y DW. Fase 3 documenta el uso analítico posterior sobre ese repositorio ya construido.
